1. 취업 준비
- 구직활동기간, 취업 준비 단계
- 포트폴리오, 입사지원서 작성
- 입사지원서 서류 중요

2. 채용시장 흐름
- "일 경험"을 가장 높게 보고 있음
- 기업에서 중요하게 여기는 요소
- : 직무 일경험 > 직무 역량 > 전공지식 (전부 다 중요)
- 챗 gpt자기소개서 등장에 따라 다른 전형 강화 가능성 증가

3. 이력서
* 서류 작성 전
- 직무 분석(비전, 핵심가치, 인재상, 직무역할, 필요역량) 
  & 자기 분석(적성, 흥미도, 강점약점, 직무 관련 경험, 직무 역량, 포트폴리오)
  사이 연결고리 찾아서 작성
- 지정양식 -> 기업 자체 양식은 수정 금지, 빈 항목 없도록 채워서 작성, 기본에 충실
- 자유양식
- > 노션, 웹 페이지
- > 사람인, 잡코리아, 점핏
- > 작성 시 파일 형식 확인 (hwp, docx, pdf 등)
- > 지원자에게 유리한 항목 많은 양식 찾을 것

- 제목 - 자신을 나타낼 수 있는 키워드를 넣어 작성
- 지원분야 - 지원하는 분야 명확히 기재할 것 (백엔드, 프론트엔드)
- 경력사항 - 기업 또는 아르바이트 등 근무 경험 있으면 근무 기간 및 담당 업무 확인 해서 기재
- 희망연봉 - 현실적으로 알아보고 적기
- 증명사진 - 깨끗하고 단정한 이미지 중요 / 정장 / ai사진 사용 절대 X(증명용으로 인정 안됨)
- 인적사항
- 학력사항, 교육/활동/경험사항
- 주요 프로젝트 (프로젝트명 / 기간 / 담당업무(맡은 업무, 역할 범위 기재) / 성과,실적)
- 보유자격 및 어학 성적(자격증 종류, 관련도(직무관련 자격증 우선 기재), 취득일 발행처 등)

* 이력서 작성법
- 자기소개서 = 지원자의 역량 확인, 미래 성과 예측
- 어떤 활동을 통해 어떤 역량을 갖추게 되었는지 기승전결 보여줘야 함
- 경험정리 - 경험을 통해 어떤 "변화" 일어났는지 파악, 무엇을 배웠는지 효과적으로 정리
- STAR 기법: 어떤 상황에서- 어떤 과제에 당면했는지 - 그래서 어떤 행동을 했는지 - 결과는 어떻게 되었는지
- SCAR 기법: 어떤 상황 - 위기 발생 - 해결하기 위해 한 행동 - 결과 
- SOARA 기법 : 상황 - 목표 - 행동 - 결과 - 영향
(어떤 상황에 직면 - 성취하고자 했던 목표 - 성취를 위해 취한 행동 - 그 결과 - 이로 인한 영향)

- 지원 동기 작성 -> SOARA (회사 가치가 나의 가치관과 일치함 보여줄 수 있어야)
- 성장 과정, 성격 장점 -> STAR, SCAR (상황 문제점 해결방법 순서 잘 드러나게 작성)
- 핵심 역량 -> SOARA(핵심 역량 발휘한 결과, 영향 잘 나타나도록)

- 작성 후 검토하기
- > 구성 깔끔한지, 주어진 양식에 빈칸 없이 잘 작성했는지, 
-   사실만 기재했는지(잘못 기재된 항목 없는지), 오탈자 없는지(맞춤법 검사기), 
-   내용이 물 흐르듯 자연스럽게 잘 읽히는지

4. 포트폴리오
- 지원자의 프로젝트 활동 및 다양한 활동 기록을 "시각적"으로 보여주는 매개체
- 가독성이 중요

- ? 포트폴리오 =/= 프로젝트
- >> 개발한 프로젝트들을 한 곳에 모아놓은 것이 포트폴리오

- ? 무조건 웹페이지로 제작할 필요 X
- >> notion, ppt 등 만들 수 있는 형태 다양함

- ? 프로젝트 동작 화면이 중요하진 않음
- >> 프로젝트 소개와 코드 및 이슈 해결방법이 더 중요

- ? 창의적인 아이디어일 필요 없음
- >> 창의적인 아이디어가 개발 수준을 향상시켜주는 것은 아님

- ** 포트폴리오 구성
- 자기소개 (이름, 자기소개(슬로건), 연락처, 이메일 등 / 사진 (증명사진 아니어도 됨. 나를 나타낼 수 있는 사진 가능))
- 슬로건 = 포폴에서 가장 먼저 보여지는 메시지. 담당자로 하여금 지원자 첫인상 결정 요소
  (슬로건에서 보여주는 메시지를 프로젝트 경험에서도 일맥상통 시켜야)
- 기술 스택 : 경험해봤고 다룰 수 있는 기술들 위주로 작성 (강의보고 따라친 수준은 X)
- 아카이빙 : 깃허브 및 블로그 링크
- 프로젝트 : 프로젝트 소개 / 기간 및 인원 / 사용 기술(개발환경) / 시스템 아키텍처** / 핵심 기능 / 성과 / 개인 기여 등
  (시스템 아키텍처 -> 백엔드는 눈에 보이지 않기 때문에 만드는 것이 좋음.
  시스템을 만들때 어떤식으로 돌아가는지에 대해 설명해주면 좋음)
- 프로젝트 -> 해당 주제 선택한 이유, 해당 기술 선택한 이유, 발생한 이슈를 해결할 때 사용한 해결 방식 선택 이유
              왜 이러한 방식을 선택했고 다른 방식은 선택하지 않았는지에 대해 대비하여 작성
